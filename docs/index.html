Features

Everything You Need for Local AI

Powerful tools and features to build, deploy, and manage AI applications entirely on your hardware.

🔒

Privacy-First

All data stays on your machine. No cloud dependencies, no data leakage. Your conversations never leave your device.

🍎

Apple Silicon Optimized

Native support for M1, M2, and M3 chips with hardware-accelerated inference. Get the best performance on your Mac.

🐳

Containerized

Docker-based deployment for consistency across environments. Easy to set up, easy to maintain.

💬

Multiple Interfaces

Choose your experience: terminal CLI, lightweight web UI, or full-featured Open WebUI with RAG support.

📚

RAG Ready

Built-in ChromaDB integration for retrieval-augmented generation. Upload documents and chat with your knowledge base.

☁️

Cloud Fallback

Optional OpenRouter integration for cloud models. Use local models by default, switch to cloud when needed.

Prerequisites

What You Need to Get Started

AI-Stack is designed to work on any Mac with sufficient resources. Here's what you need:

💻 Hardware Requirements

  • ✓
    Mac with Apple Silicon (M1/M2/M3/M4) — Recommended for best performance
  • ✓
    8GB RAM minimum — 16GB recommended for larger models
  • ✓
    20GB free disk space — For models and Docker images
  • ⚠
    Intel Mac — Works but significantly slower (2-3x)

🔧 Software Requirements

  • ✓
    macOS 12+ — Monterey or later
  • ✓
    OrbStack — Lightweight Docker alternative (recommended)
    brew install orbstack
  • ✓
    Docker Desktop — Alternative to OrbStack
    brew install docker
  • ✓
    Ollama — For local model inference
    brew install ollama

📦 Optional but Recommended

  • ✓
    OpenRouter API Key — For cloud model access
    Get free key →
  • ✓
    Homebrew — Package manager for easy installation
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Quick Installation Script 
# Install all prerequisites on macOS
# 1. Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. Install OrbStack (Docker alternative)
brew install orbstack

# 3. Install Ollama
brew install ollama

# 4. Clone and setup AI-Stack
git clone https://github.com/irfancode/ai-stack.git
cd ai-stack
./scripts/setup-lean.sh
./scripts/setup.sh
Quick Start

Get Running in Minutes

Three simple steps to your local AI setup.

Terminal 
# 1. Clone or download the repository
git clone https://github.com/irfancode/ai-stack.git
cd ai-stack

# 2. Run the lean setup (downloads models)
./scripts/setup-lean.sh

# 3. Start the full stack
./scripts/setup.sh

Models

Choose the Right Model

Different models for different needs. All run locally on your machine.

llama3.2:3b

Size: ~2GB Speed: ~30 tok/s

Balanced model with good general-purpose performance. Excellent for everyday tasks, coding, and creative writing. Best value for 8GB systems.

Best for: General Chat Coding Creative Writing

qwen3:1b

Size: ~1.2GB Speed: ~50 tok/s

Ultra-fast model for quick tasks. Perfect for constrained hardware or when you need rapid responses. Great for scripts and automation.

Best for: Quick Tasks Scripts Low RAM

phi3:3.8b-mini

Size: ~2.4GB Speed: ~25 tok/s

Microsoft's efficient model with excellent reasoning capabilities. Better at complex tasks and nuanced understanding. Good quality/performance balance.

Best for: Reasoning Analysis Instructions

mistral:7b

Size: ~4GB Speed: ~15 tok/s

Higher capability model for more complex tasks. Requires more RAM but provides better quality responses. Best for nuanced conversations.

Best for: Detailed Analysis Long Context 16GB+ RAM

codellama:7b

Size: ~4GB Speed: ~15 tok/s

Specialized for code. Understands programming patterns, can explain code, write tests, and suggest improvements. Great for developers.

Best for: Code Generation Bug Fixes Code Review

llama3.2:1b

Size: ~1.3GB Speed: ~60 tok/s

Smallest capable model. Extremely fast, minimal RAM usage. Good for simple tasks, commands, and systems with very limited resources.

Best for: Minimal RAM Simple Tasks Commands

Troubleshooting

Common Issues & Solutions

Quick fixes for frequent problems.

⚠️ Out of Memory

Model won't load or system slows down

# Use smaller models
ollama pull llama3.2:3b
ollama rm mistral:7b

# Add to ~/.zshrc for memory optimization
export OLLAMA_MAX_LOADED_MODELS=1
export OLLAMA_NUM_PARALLEL=1

⚠️ Port Already in Use

Services can't start

# Check what's using the port
lsof -i:3000
lsof -i:8000

# Kill the process or stop the service
docker compose down
pkill ollama

⚠️ Services Won't Start

Docker containers fail to launch

# Check Docker is running
docker info

# Restart Docker
# (System Preferences > Docker > Restart)

# Validate configuration
./scripts/validate.sh

# Fresh start
docker compose down -v
./scripts/setup.sh

⚠️ Can't Login to WebUI

Authentication issues

# Reset to fresh install
docker compose down -v
rm .env
./scripts/setup.sh

# Or check your password
grep ADMIN_PASSWORD .env

⚠️ Slow Performance

Models run slowly

# Use Apple Silicon optimized models
ollama pull llama3.2:3b  # Native ARM

# Check system resources
./scripts/health.sh

# Enable flash attention (in ~/.zshrc)
export OLLAMA_FLASH_ATTENTION=1

⚠️ Model Not Found

Can't run a specific model

# List installed models
ollama list

# Pull a model
ollama pull llama3.2:3b

# Check available models
ollama list
